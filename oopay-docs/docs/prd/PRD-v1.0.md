# OOPay 产品需求文档 (PRD) v1.0

## 1. 产品概述

### 1.1 产品定位
OOPay 是企业级聚合支付平台，为商户提供统一接口对接多支付通道，支持智能路由、风险控制和自动化对账。

### 1.2 核心能力
- 多通道聚合：微信、支付宝、USDT 等
- 智能路由：基于成功率/费率/延迟自动选择
- 资金安全保障：Saga 事务 + 多级对账

### 2.1 包含模块
| 模块 | 说明 |
|------|------|

### 2.2 剔除模块

- ❌ 代理商系统 (agent)

---

## 3. 核心业务流程

### 3.1 支付流程
```
商户请求 → 参数校验 → 创建订单 → 路由选择 → 调用通道 → 返回参数 → 异步通知
```

**关键规则：**
- 订单有效期：30 分钟
- 幂等窗口：24 小时（同商户订单号）
- 异步通知：最多 5 次，间隔 1/2/4/8/16 分钟

### 3.2 退款流程
```
退款申请 → 订单校验 → 资金冻结 → 通道退款 → 更新订单 → 解冻资金 → 异步通知
```

**关键规则：**
- 仅支持全额退款
- 退款金额原路返回
- 退款订单号独立生成

## 5. 数据一致性保障

| 层级 | 措施 |
|------|------|
| 接口层 | 幂等 Token、请求去重 |
| 业务层 | Saga 事务、状态机校验 |
| 数据层 | 乐观锁/悲观锁、唯一索引 |
| 缓存层 | Redis 原子操作、缓存一致性 |
| 对账层 | 多维度对账、差异告警 |
| 监控层 | 数据一致性巡检、异常订单发现 |

---

## 6. 安全要求

| 项 | 要求 |
|----|------|
| 签名 | HMAC-SHA256，含 timestamp + nonce |
| 防重放 | 5 分钟窗口 + nonce 唯一性 |
| 限流 | 单商户 100 req/s |
| 加密 | AES-256 敏感数据，bcrypt 密码 |
| 大额确认 | TG Bot 二次确认 (>10000 USDT) |
