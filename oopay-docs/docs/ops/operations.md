# OOPay 运营手册

## 1. 日常运维

### 1.1 每日检查清单

| 时间 | 检查项 | 命令/方式 | 正常指标 |
|------|--------|-----------|----------|
| 09:00 | 系统健康 | 查看监控大盘 | 所有服务 UP |
| 09:30 | 支付成功率 | 查看昨日报表 | > 99.5% |
| 10:00 | 异常订单 | 查询异常订单列表 | 无未处理异常 |
| 14:00 | 通道状态 | 检查通道健康状态 | 所有通道正常 |
| 17:00 | 日终对账 | 查看对账结果 | 无差异或差异已处理 |
| 20:00 | 资源使用 | 查看 CPU/内存/磁盘 | < 70% |

### 1.2 监控指标

**关键业务指标：**
- 支付成功率：目标 > 99.5%
- 平均支付耗时：目标 < 3s
- 退款成功率：目标 > 99%
- 对账差异率：目标 < 0.01%

**关键系统指标：**
- API 响应时间 P99：< 500ms
- 数据库连接池使用率：< 70%
- Redis 内存使用率：< 80%
- 错误率：< 0.1%

## 2. 故障处理

### 2.1 故障分级

| 级别 | 定义 | 响应时间 | 通知方式 |
|------|------|----------|----------|
| P0 | 支付不可用 | 5分钟 | 电话+短信+钉钉 |
| P1 | 单通道故障 | 15分钟 | 短信+钉钉 |
| P2 | 性能下降 | 30分钟 | 钉钉 |
| P3 | 一般告警 | 1小时 | 钉钉 |

### 2.2 常见故障处理

**支付成功率下降：**
1. 检查各通道成功率
2. 定位问题通道
3. 临时切换备用通道
4. 联系通道方排查

**数据库连接池耗尽：**
1. 检查慢查询
2. 临时扩容连接池
3. 优化 SQL 或加索引
4. 考虑读写分离

**Redis 故障：**
1. 检查集群状态
2. 触发主从切换
3. 检查网络连接
4. 必要时重启实例

**回调堆积：**
1. 检查通知队列长度
2. 检查回调地址可用性
3. 临时增加通知线程
4. 处理死信消息

### 2.3 应急预案

**支付服务不可用：**
```
1. 立即启用降级模式
2. 暂停新订单接入
3. 保留已有订单处理
4. 通知商户暂停收款
5. 全力恢复服务
```

**资金异常：**
```
1. 立即冻结相关账户
2. 停止所有出金操作
3. 核对账务差异
4. 联系技术负责人
5. 保留审计日志
```

## 3. 通道管理

### 3.1 通道上线流程

1. **测试环境验证**
   - 接口连通性测试
   - 全流程支付测试
   - 回调接收测试
   - 退款流程测试

2. **配置生产参数**
   - 配置商户号/密钥
   - 设置费率
   - 配置限额
   - 设置权重

3. **灰度发布**
   - 先开放 1% 流量
   - 监控成功率
   - 逐步放大流量
   - 全量发布

### 3.2 通道下线流程

1. 关闭新订单路由
2. 等待存量订单完成
3. 确认无未处理回调
4. 标记通道状态为下线

### 3.3 费率调整

| 场景 | 操作 | 生效时间 |
|------|------|----------|
| 新商户 | 按默认费率 | 立即 |
| 商户调价 | 修改商户通道配置 | 立即 |
| 通道调价 | 修改通道费率 | 次日0点 |

## 4. 对账管理

### 4.1 日终对账流程

1. **自动下载对账单**（凌晨 2:00）
2. **解析对账单文件**
3. **逐笔比对订单**
4. **生成差异报告**
5. **发送通知给运营**

### 4.2 差异处理

| 差异类型 | 处理方式 | 负责人 |
|----------|----------|--------|
| 系统有通道无 | 自动关闭订单 | 系统 |
| 通道有系统无 | 自动补单 | 系统 |
| 金额不符 | 人工核实 | 运营 |
| 状态不符 | 以通道为准修正 | 运营 |

### 4.3 对账异常处理

**对账单下载失败：**
1. 检查通道 SFTP/接口
2. 手动下载对账单
3. 重新触发对账

**对账差异过大：**
1. 停止自动处理
2. 人工核查原因
3. 批量修正或回滚

## 5. 商户管理

### 5.1 商户入驻审核

**审核要点：**
- 营业执照真实性
- 法人身份证有效性
- 经营范围合规性
- 网站/App 内容审核

**审核结果：**
- 通过：发送 API 密钥
- 拒绝：说明拒绝原因
- 补充材料：通知补充

### 5.2 风险商户处置

| 风险等级 | 处置措施 |
|----------|----------|
| 低风险 | 关注，增加抽查频率 |
| 中风险 | 限制单笔/日限额 |
| 高风险 | 冻结账户，人工审核 |

### 5.3 商户问题处理

**投诉处理流程：**
1. 记录投诉内容
2. 查询相关订单
3. 核实交易情况
4. 联系商户沟通
5. 给出处理方案
6. 跟进处理结果

## 6. 系统维护

### 6.1 定期维护

| 周期 | 内容 | 时间 |
|------|------|------|
| 每日 | 数据备份 | 凌晨 3:00 |
| 每周 | 日志清理 | 周日 4:00 |
| 每月 | 索引优化 | 月初 5:00 |
| 每季 | 全量备份 | 季度首月 |

### 6.2 发布管理

**发布窗口：**
- 常规发布：周三/周四 凌晨 2:00-4:00
- 紧急发布：随时，需审批

**发布流程：**
1. 提交发布申请
2. 代码评审通过
3. 测试环境验证
4. 预发布环境验证
5. 生产灰度发布
6. 监控观察
7. 全量发布或回滚

### 6.3 数据备份

**备份策略：**
- MySQL：每日全量 + Binlog 增量
- Redis：RDB 每小时 + AOF 实时
- 配置：版本控制 + 定期导出

**备份验证：**
- 每月恢复演练
- 验证数据完整性
- 记录恢复时间
