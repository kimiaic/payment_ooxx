### 1.2 核心服务职责

| 服务名 | 职责 | 关键技术 |
|--------|------|----------|
| oopay-gateway | 支付请求入口，统一验签、限流、路由 | Spring Cloud Gateway, Lua |
| oopay-order | 订单生命周期管理，状态机驱动 | Spring State Machine |
| oopay-payment | 支付核心，调用通道、处理回调 | Saga 模式 |
| oopay-merchant | 商户管理、密钥管理、配置管理 | - |
| oopay-account | 资金账户、余额、流水 | 乐观锁 |
| oopay-router | 智能路由决策 | Drools / 自定义算法 |
| oopay-channel | 通道适配器集群 | 策略模式 |
| oopay-reconcile | 对账引擎，实时/定时/日终 | XXL-JOB |
| oopay-risk | 风控规则引擎 | Drools |
| oopay-notify | 异步通知，支持多渠道 | 消息队列 |

**关键说明：**
- 步骤②：网关层完成所有前置校验，失败直接返回，不进入业务层
- 步骤④：订单创建即落库，状态=PENDING，生成全局唯一订单号
- 步骤⑥：路由引擎综合考虑商户配置、通道费率、历史成功率
- 步骤⑩：返回的支付参数（如 prepay_id）需加密后返回给商户


### 3.2 故障切换机制

```
通道健康检查（每 30 秒）：
  • 探测接口可用性
  • 统计最近 5 分钟成功率
  • 平均响应时间

故障判定：
  • 连续失败次数 > 3
  • 或 5 分钟成功率 < 50%

切换策略：
  • 标记通道状态为 DEGRADED
  • 触发告警通知
  • 新订单路由到备用通道
  • 10 分钟后自动恢复探测
```

### 4.2 USDT 汇率计算

```
商户显示汇率 = 基础汇率 × (1 + 点差率) × (1 + 手续费率)

示例：
  基础汇率: 1 USDT = 7.2500 CNY
  点差率: 0.3%
  手续费率: 0.5%
  
  计算: 7.2500 × 1.003 × 1.005 = 7.3034 CNY
```

## 6. 监控告警

### 6.1 监控指标体系

```
业务指标：
  • 订单成功率 (目标: > 99.5%)
  • 平均支付耗时 (目标: < 3s)
  • 各通道成功率

系统指标：
  • API 响应时间 P99 (目标: < 500ms)
  • 错误率 (目标: < 0.1%)
  • QPS / 吞吐量

资源指标：
  • CPU 使用率 (告警: > 70%)
  • 内存使用率 (告警: > 80%)
  • 磁盘使用率 (告警: > 85%)
  • 数据库连接池使用率 (告警: > 70%)
```

### 6.2 告警分级

| 级别 | 触发条件 | 通知方式 | 响应时间 |
|------|----------|----------|----------|
| P0 | 支付成功率 < 95% | 电话 + 短信 + 钉钉 | 5分钟内 |
