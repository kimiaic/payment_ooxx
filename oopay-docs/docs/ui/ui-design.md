# OOPay UI 设计文档

## 1. 设计原则

### 1.1 视觉风格
- **色彩**：主色深蓝 #1890ff，辅色绿色 #52c41a（成功）、红色 #ff4d4f（危险）
- **字体**：系统默认，中文优先微软雅黑
- **布局**：侧边栏导航 + 内容区，响应式适配

### 1.2 交互原则
- 操作反馈：3 秒内响应
- 错误提示：明确告知原因和解决方案
- 数据展示：敏感信息脱敏（手机号、卡号）

## 4. 组件规范

### 4.1 状态标签

| 状态 | 颜色 | 示例 |
|------|------|------|
| 成功/正常/启用 | 绿色 #52c41a | [● 成功] |
| 处理中/待审核 | 黄色 #faad14 | [● 处理中] |
| 失败/禁用 | 红色 #ff4d4f | [● 失败] |
| 关闭/过期 | 灰色 #bfbfbf | [● 关闭] |

### 4.2 按钮规范

| 类型 | 样式 | 用途 |
|------|------|------|
| 主要 | 蓝色实心 | 提交、保存、确认 |
| 次要 | 白色边框 | 取消、返回 |
| 危险 | 红色实心 | 删除、禁用、关闭 |
| 链接 | 蓝色文字 | 详情、编辑 |

### 4.3 表单校验

```
校验时机：
  • 失焦时：格式校验（手机号、邮箱等）
  • 提交时：必填校验、业务校验

错误提示：
  • 位置：字段下方红色文字
  • 内容：明确告知错误原因
  • 示例："商户编号不能为空"、"金额必须大于0"
```

### 5.1 断点

| 设备 | 断点 | 布局调整 |
|------|------|----------|
| 桌面 | > 1200px | 完整侧边栏 |
| 平板 | 768-1200px | 折叠侧边栏 |
| 手机 | < 768px | 底部导航 |
